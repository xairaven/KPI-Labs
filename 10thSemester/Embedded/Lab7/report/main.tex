	\section*{Хід роботи.}
	
	Виконання сьомої лабораторної роботи було присвячено глибокому дослідженню принципів роботи та програмного керування символьним рідкокристалічним дисплеєм на базі промислового стандарту HD44780. Оскільки дана лабораторна робота є логічним продовженням попередньої, апаратно-програмна модель розроблялася у середовищі SimulIDE на базі мікроконтролера ATmega32 (тактова частота 8 МГц) у комплексі з цифровим датчиком температури DS18B20.
	
	\image{10cm}{scheme}
	
	Фізична схема підключення компонентів залишилася ідентичною до схеми з шостої лабораторної роботи: датчик температури підключено до лінії PA0 з підтягуючим резистором 4.7 кОм, а LCD-дисплей підключено до порту С мікроконтролера. Проте головним об'єктом дослідження цього разу став саме контролер дисплея HD44780 та низькорівневі механізми взаємодії з ним. У попередній роботі дисплей відігравав роль пасивного інструменту для виведення результатів роботи шини 1-Wire, тоді як у цій роботі датчик температури виступав лише генератором вхідних даних для тестування повноцінного програмного драйвера екрана.
	
	Для економії виводів мікроконтролера дисплей було підключено в 4-бітному режимі керування. Інформаційні лінії D4-D7 дисплея з'єднані з пінами PC4-PC7, лінія вибору регістрів RS -- з PC0, а лінія стробування EN -- з PC1. Лінія RW (читання/запис) була жорстко підключена до загального дроту (GND), оскільки в даній моделі передбачено лише запис даних у пам'ять дисплея. 
	
	Ключові зміни відбулися у програмному коді, який було доповнено розширеним функціоналом для повноцінного керування пам'яттю HD44780. Програмний драйвер дисплея реалізує жорстку послідовність ініціалізації при подачі живлення, надсилаючи специфічні шістнадцяткові коди (\texttt{0x30}, \texttt{0x20}) для переходу в 4-бітний режим. Оскільки дані передаються по чотирьох лініях, кожен 8-бітний байт (команда або символ) розбивається на дві тетради (старшу та молодшу), які передаються послідовно із супроводжуючим стробуючим імпульсом на лінії EN тривалістю близько 50 мікросекунд. Далі дисплей налаштовується командою \texttt{0x28} на дворядковий режим відображення зі шрифтом 5x8 пікселів, вмикається командою \texttt{0x0C} (без відображення курсора) та очищується командою \texttt{0x01}.
	
	Головним нововведенням у коді стала реалізація функції позиціювання курсора \texttt{LCD\_SetCursor}. На відміну від попередньої роботи, де текст виводився суцільним потоком, тепер мікроконтролер звертається до конкретних адрес пам'яті дисплея (DDRAM). Для переходу на перший рядок використовується базова адреса \texttt{0x80}, а для другого -- \texttt{0xC0}. Завдяки цьому алгоритм головного циклу було суттєво перероблено: програма спочатку очищує екран, встановлює курсор на початок першого рядка і виводить статичний текст <<Temperature:>>, після чого переводить курсор на другий рядок і виводить відформатоване значення температури зі знаком та дробовою частиною. 
	
	У результаті моделювання було отримано стабільну систему, яка демонструє повний програмний контроль над матричним LCD-дисплеєм. Зміна температури на датчику DS18B20 миттєво обробляється мікроконтролером, а результати коректно і структуровано оновлюються у відповідних комірках пам'яті дисплея, що підтверджує успішне виконання завдань лабораторної роботи.
	
	\subsection*{Висновок}
	
	Під час виконання сьомої лабораторної роботи було успішно досліджено принципи роботи та програмного керування символьним рідкокристалічним дисплеєм на базі контролера HD44780. Практично реалізовано діючу апаратно-програмну модель у середовищі SimulIDE, де мікроконтролер ATmega32 забезпечує зчитування даних з датчика температури та їх виведення на LCD-екран у 4-бітному режимі керування. У ході роботи було створено повноцінний програмний драйвер дисплея мовою С, який реалізує жорстку послідовність ініціалізації, генерування стробуючих імпульсів, вибір регістрів команд та даних, а також точне позиціювання курсора у пам'яті DDRAM. Це дозволило організувати структуроване та зручне для користувача відображення статичної і динамічної інформації, підтвердивши високу ефективність використання LCD-індикаторів у вбудованих системах.
	
	\section*{Контрольні запитання.}
	
	\begin{enumerate}
		\item \textbf{Де використовуються рідкокристалічні індикатори?} \\ Рідкокристалічні індикатори (РКІ) широко використовуються для відображення графічної або текстової інформації в ноутбуках, телефонах, цифрових фотоапаратах, електронних книгах, навігаторах, планшетах, електронних перекладачах, калькуляторах, годинниках та у різноманітних вбудованих системах для відображення інформації про результати контролю.
		
		\item \textbf{Назвіть основні характеристики мікросхеми HD44780.} \\ HD44780 є промисловим стандартом контролера алфавітно-цифрових РКІ. Він здатний керувати дисплеями, що відображають символи у вигляді матриці 5x8 або 5x10 пікселів. Контролер підтримує паралельні 8-бітний та 4-бітний інтерфейси передачі даних і містить вбудовану пам'ять даних дисплея (DDRAM) та пам'ять генератора символів (CGRAM).
		
		\item \textbf{Назвіть призначення виводів індикатора типу HD44780.} \\ Індикатор має лінії живлення та контрастності (VDD, VSS, V0), керуючі лінії та шину даних. Лінія RS (Register Select) вибирає регістр команд (0) або даних (1). Лінія RW (Read/Write) визначає напрямок передачі: запис (0) або читання (1). Лінія EN (Enable) слугує для подачі стробуючого імпульсу, по спаду якого фіксуються дані. Лінії D0-D7 утворюють двонаправлену шину даних, з яких у 4-бітному режимі використовуються лише чотири старші (D4-D7).
		
		\item \textbf{Назвіть та опишіть команди керування індикатором типу HD44780.} \\ До основних команд належать: Clear Display (\texttt{0x01}) -- повністю очищує екран; Return Home (\texttt{0x02}) -- повертає курсор на початкову позицію; Display Control (наприклад, \texttt{0x0C}) -- вмикає дисплей та керує відображенням/блиманням курсора; Function Set (наприклад, \texttt{0x28}) -- налаштовує розрядність інтерфейсу (4 або 8 біт), кількість рядків та шрифт; Set DDRAM Address (наприклад, \texttt{0x80} або \texttt{0xC0}) -- переміщує курсор у вказану комірку пам'яті для виведення наступного символу.
		
		\item \textbf{Наведіть та опишіть схему підключення індикатора типу HD44780 в складі моделі цифрового вольтметра.} \\ Хоча методичні вказівки можуть наводити приклад вольтметра, у нашій практичній моделі системи вимірювання температури LCD підключено за аналогічною універсальною 4-бітною схемою. Керуючі лінії RS та EN підключені до пінів PC0 та PC1 мікроконтролера ATmega32 відповідно. Лінія RW жорстко заземлена (GND), оскільки відбувається лише запис даних. Шина даних дисплея (D4-D7) підключена до старшої тетради порту С (PC4-PC7), що дозволяє суттєво економити вільні виводи мікроконтролера.
		
		\item \textbf{Наведіть та опишіть схему алгоритму роботи пристрою керування LCD-дисплеєм.} \\ Алгоритм починається з ініціалізації портів вводу-виводу на вихід. Далі виконується обов'язкова послідовність апаратної ініціалізації дисплея: подаються команди переходу в 4-бітний режим (\texttt{0x30}, \texttt{0x20}), налаштування шрифту та кількості рядків (\texttt{0x28}), вимкнення курсора (\texttt{0x0C}) і повного очищення екрана (\texttt{0x01}). У головному робочому циклі мікроконтролер встановлює курсор на перший рядок (базова адреса \texttt{0x80}) для виведення статичного тексту, а потім переводить курсор на другий рядок (базова адреса \texttt{0xC0}) для посимвольного виведення буфера зі згенерованим результатом вимірювання.
		
		\item \textbf{Наведіть та опишіть робочу програму пристрою керування LCD-дисплеєм.} \\ Робоча програма мовою С містить набір низькорівневих функцій-драйверів. Функція \texttt{LCD\_Pulse()} формує короткий стробуючий імпульс на лінії EN. Функція \texttt{LCD\_Send()} приймає байт даних та прапорець режиму (RS), розбиває байт на дві тетради та по черзі відправляє їх у порт. Функція \texttt{LCD\_Init()} конфігурує мікросхему при старті системи, а \texttt{LCD\_SetCursor()} вираховує адресу пам'яті (DDRAM) залежно від заданого рядка та колонки. Функція \texttt{main()} керує загальним потоком, форматуванням тексту за допомогою \texttt{sprintf} та його передачею драйверу.
		
		\item \textbf{Поясніть використання LCD-індикатора в моделі мережі 1-WIRE.} \\ У моделі мікроконтролерної мережі 1-Wire LCD-індикатор виконує роль головного людино-машинного інтерфейсу. На відміну від 7-сегментних дисплеїв, LCD дозволяє виводити розшифровані цифрові дані з датчика DS18B20 у зручному текстовому форматі, чітко розмежовуючи статичну інформацію (підписи) та динамічні дані (значення температури з десятковими дробами та символом знаку). Це робить систему візуально інформативною, більш гнучкою у налаштуваннях та зручною для користувача.
	\end{enumerate}

	\section*{Лістинг.}
	
	\textbf{main.c}
	
	\begin{minted}{c} 
		#define F_CPU 8000000UL
		#include <avr/io.h>
		#include <util/delay.h>
		#include <stdio.h>
		
		// ================= LCD DRIVER (HD44780) =================
		#define LCD_PORT PORTC
		#define LCD_DDR DDRC
		#define RS PC0
		#define EN PC1
		
		// Strobe pulse generation function (write data to display)
		void LCD_Pulse(void) {
			LCD_PORT |= (1 << EN);   // Set EN to 1
			_delay_us(1);            // Short delay
			LCD_PORT &= ~(1 << EN);  // Reset EN to 0 (data is latched on falling edge)
			_delay_us(50);
		}
		
		// Function to send a byte (command or data) in 4-bit mode
		void LCD_Send(unsigned char value, unsigned char mode) {
			if (mode) LCD_PORT |= (1 << RS); // Data mode (RS = 1)
			else LCD_PORT &= ~(1 << RS);     // Command mode (RS = 0)
			
			// Send high nibble (4 bits)
			LCD_PORT = (LCD_PORT & 0x0F) | (value & 0xF0);
			LCD_Pulse();
			
			// Send low nibble (4 bits)
			LCD_PORT = (LCD_PORT & 0x0F) | ((value << 4) & 0xF0);
			LCD_Pulse();
			_delay_ms(2); // Command execution delay
		}
		
		// Display initialization function
		void LCD_Init(void) {
			LCD_DDR = 0xFF; // Configure Port C as output
			_delay_ms(20);  // Wait for display power stabilization
			
			// Mandatory reset sequence for HD44780
			LCD_PORT = (LCD_PORT & 0x0F) | 0x30;
			LCD_Pulse(); _delay_ms(5);
			LCD_Pulse(); _delay_us(150);
			LCD_Pulse(); _delay_us(150);
			
			// Switch display to 4-bit mode
			LCD_PORT = (LCD_PORT & 0x0F) | 0x20; 
			LCD_Pulse(); _delay_ms(2);
			
			// Parameter configuration
			LCD_Send(0x28, 0); // 0x28: 4-bit interface, 2 lines, 5x8 font
			LCD_Send(0x0C, 0); // 0x0C: Display ON, cursor OFF (not blinking)
			LCD_Send(0x01, 0); // 0x01: Clear screen completely
			_delay_ms(2);
		}
		
		// Function to print a string of text
		void LCD_String(char* str) {
			while (*str) {
				LCD_Send(*str++, 1); // Send each character one by one
			}
		}
		
		// Function to set cursor (col - column 0..15, row - line 0..1)
		void LCD_SetCursor(unsigned char col, unsigned char row) {
			// 0x80 - base address of the first line, 0xC0 - second line
			unsigned char address = (row == 0) ? (0x80 + col) : (0xC0 + col);
			LCD_Send(address, 0);
		}
		
		// ================= 1-WIRE DRIVER (DS18B20) =================
		#define DQ_PIN PA0
		#define DQ_PORT PORTA
		#define DQ_DDR DDRA
		#define DQ_PIN_REG PINA
		
		unsigned char W1_Init(void) {
			unsigned char state;
			DQ_DDR |= (1 << DQ_PIN);
			DQ_PORT &= ~(1 << DQ_PIN);
			_delay_us(480);
			DQ_DDR &= ~(1 << DQ_PIN);
			DQ_PORT &= ~(1 << DQ_PIN);
			_delay_us(70);
			state = (DQ_PIN_REG & (1 << DQ_PIN)) ? 1 : 0;
			_delay_us(410);
			return state;
		}
		
		void W1_WriteByte(unsigned char data) {
			unsigned char i;
			for (i = 0; i < 8; i++) {
				DQ_DDR |= (1 << DQ_PIN);
				DQ_PORT &= ~(1 << DQ_PIN);
				_delay_us(2);
				if (data & (1 << i)) DQ_DDR &= ~(1 << DQ_PIN);
				_delay_us(60);
				DQ_DDR &= ~(1 << DQ_PIN);
				_delay_us(2);
			}
		}
		
		unsigned char W1_ReadByte(void) {
			unsigned char i, data = 0;
			for (i = 0; i < 8; i++) {
				DQ_DDR |= (1 << DQ_PIN);
				DQ_PORT &= ~(1 << DQ_PIN);
				_delay_us(2);
				DQ_DDR &= ~(1 << DQ_PIN);
				_delay_us(10);
				if (DQ_PIN_REG & (1 << DQ_PIN)) data |= (1 << i);
				_delay_us(50);
			}
			return data;
		}
		
		// ================= MAIN PROGRAM =================
		int main(void) {
			unsigned char t_low, t_high;
			int temp16;
			unsigned char temp_int;
			unsigned int frac_decimal;
			char buffer[16];
			
			// Initialize display
			LCD_Init();
			
			// Beautiful startup screen (LCD demonstration)
			LCD_SetCursor(1, 0); // Offset 1 character, 1st line
			LCD_String("Lab 7: LCD");
			LCD_SetCursor(0, 1); // Start of 2nd line
			LCD_String("Initializing...");
			_delay_ms(1500);
			
			while (1) {
				if (W1_Init() == 0) { 
					// Start temperature conversion
					W1_WriteByte(0xCC); 
					W1_WriteByte(0x44); 
					_delay_ms(750);     
					
					// Read result
					W1_Init();
					W1_WriteByte(0xCC); 
					W1_WriteByte(0xBE); 
					
					t_low = W1_ReadByte();  
					t_high = W1_ReadByte(); 
					temp16 = (t_high << 8) | t_low; 
					
					// Clear screen before new output
					LCD_Send(0x01, 0); 
					
					// Print static text on the first line
					LCD_SetCursor(0, 0);
					LCD_String("Temperature:");
					
					// Move to the second line to print the value
					LCD_SetCursor(0, 1);
					
					if ((temp16 & 0x8000) == 0) {
						LCD_Send('+', 1); // '+' character
					} else {
						LCD_Send('-', 1); // '-' character
						temp16 = ~temp16 + 1; 
					}
					
					temp_int = (unsigned char)(temp16 >> 4);
					frac_decimal = ((temp16 & 0x0F) * 10) / 16; 
					
					// Formatting and output
					sprintf(buffer, "%d.%d C", temp_int, frac_decimal);
					LCD_String(buffer);
					
				} else {
					LCD_Send(0x01, 0);
					LCD_String("Sensor Error!");
				}
				
				_delay_ms(500);
